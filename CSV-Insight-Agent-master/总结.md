```text
一个基于 LangChain Agent Loop + Tool Calling + Memory 的 CSV 数据分析智能体。
```

项目能展示你会：

- 用 LLM 做真实任务自动化；
- 用 LangChain / ReAct-style Agent Loop 组织工具调用；
- 把数据分析能力封装成 Skill / Tool；
- 用 MemoryStore 记录历史任务和图表偏好；
- 生成可展示的图表、Markdown、HTML、PDF 报告；
- 写 Streamlit UI、CLI、测试和 README。

## 2. 它适不适合大二找暑假实习？

### 2.1 适合的原因

这个项目适合大二学生找实习，主要是因为它有三个优点：

### 第一，项目难度不低，但还能讲清楚

很多大二项目容易出现两个极端：

1. 太简单：比如普通管理系统、爬虫、静态网页、简单聊天机器人；
2. 太大：比如完整多租户平台、复杂微服务、企业级数据平台。

`CSV-Insight-Agent` 处在比较好的中间位置。

它的业务目标很直观：

```text
上传 CSV -> 自动分析 -> 自动画图 -> 自动生成洞察 -> 自动导出报告
```

面试官一听就能理解项目在做什么，不需要你解释很久。

但它的技术实现又不算浅：

- 有 Agent Loop；
- 有 Tool Calling；
- 有 Skill Registry；
- 有 Memory；
- 有报告导出；
- 有多模式工作流；
- 有测试；
- 有前端展示。

这对大二找实习来说已经比较有竞争力。

### 第二，它符合当前 AI 应用实习的热门方向

现在很多 AI 应用相关实习岗位会关注这些关键词：

- LLM；
- LangChain；
- Agent；
- Tool Calling；
- RAG / Memory；
- 数据分析自动化；
- Prompt Engineering；
- Python 工程能力；
- 可视化和报告生成。

这个项目虽然不是 RAG 项目，但它覆盖了很多 Agent 应用方向的关键词。

尤其是现在项目已经改成：

```text
LangChain Agent Loop + LangChain Tools + SkillRegistry + MemoryStore
```

这比单纯写：

```text
调用大模型生成图表解释
```

要高级很多。

### 第三，它有真实可展示产物

这个项目不是只有命令行输出文字，而是能生成：

- PNG 图表；
- 中文分析洞察；
- Markdown 报告；
- HTML 报告；
- PDF 报告；
- Agent 执行过程 Trace。

这对实习非常重要。

因为面试时你可以直接演示：

```text
上传一个 sales.csv，输入“生成销售数据分析报告”，系统自动分析字段、选择图表、生成洞察和报告。
```

这种演示比口头讲“我做了一个 Agent”更有说服力。

```text
CSV-Insight-Agent 是一个基于 LangChain Agent Loop、Tool Calling 和 MemoryStore 的 CSV 数据分析智能体。用户上传 CSV 并输入自然语言目标后，Agent 会自动调用数据画像、图表推荐、图表生成、洞察总结和报告导出等工具，最终生成 PNG 图表、中文分析结论以及 Markdown / HTML / PDF 报告。
```

- 有最终产物；
- 面试官容易理解。

## 6. 面试时怎么讲这个项目

建议按下面这个逻辑讲，不要一上来堆技术名词。

### 6.1 先讲动机

```text
很多 CSV 分析工具需要用户自己选字段、选图表、写分析结论。我想做一个轻量的数据分析 Agent，让用户只上传 CSV 和输入目标，系统就能自动完成数据画像、图表选择、图表生成、洞察总结和报告导出。
```

### 6.2 再讲架构

```text
项目分成四层：
第一层是 Streamlit UI 和 CLI；
第二层是 LangChain Agent Loop 和 LangGraph stable workflow；
第三层是 SkillRegistry，把数据画像、图表生成、报告导出等能力封装成工具；
第四层是 MemoryStore，用 JSONL / JSON 保存历史任务和图表偏好。
```

### 6.3 再讲 Agent Loop

```text
在 agent_loop 模式下，LLM 不直接生成代码，而是通过 LangChain Tool 调用白名单工具。它会先调用 profile_csv 获取字段和统计信息，再调用 suggest_chart 选择图表，然后调用 plot_chart_batch 生成图表，最后调用 generate_insight 和 export_pdf 生成报告。
```

### 6.4 最后讲安全和工程化

```text
为了避免 LLM 生成任意 Python 代码，我只允许它调用注册过的 Skill。图表生成也是参数化执行，不让模型直接执行代码。同时我写了测试覆盖工具注册、LangChain Tool wrapper、Agent Runner、MemoryStore 和路由逻辑。
```

这个讲法会比单纯说“我用了 LangChain”更有质量。

## 8. 目前还存在的不足

虽然这个项目已经适合作为暑假实习项目，但如果想让它更强，还可以继续补几项。

### 8.1 Agent Loop 的稳定性还可以继续增强

当前项目已经有 `agent_loop`，但真实 LLM 多步工具调用可能会受到模型输出格式影响。

可以继续优化：

- 更严格的工具参数 Schema；
- 更清晰的 ReAct prompt；
- 工具调用失败后的自动重试；
- 展示完整 Agent Trace；
- 给每一步增加错误恢复策略。

### 8.2 记忆模块还比较轻量

现在 MemoryStore 是文件型 JSONL / JSON 记忆，适合轻量项目，但不是特别高级。

如果时间充足，可以加：

- 向量记忆；
- 用户偏好总结；
- 自动根据历史任务推荐图表风格；
- 相似 CSV 任务检索。

但注意：找暑假实习不一定需要加太复杂，别把项目做爆。

### 8.3 缺少部署演示

如果面试时能给一个在线地址，会更有说服力。

可以考虑：

- 部署到 Streamlit Community Cloud；
- 或者录一个 1 分钟 demo GIF；
- 或者在 README 里放截图和示例报告。

这比继续堆新功能更划算。
