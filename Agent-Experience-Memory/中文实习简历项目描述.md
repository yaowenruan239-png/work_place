# Agent Experience Memory Engine｜面向 AI Agent 的经验记忆检索系统

## 一句话项目简介

设计并实现面向 AI Agent 的经验记忆检索系统，将 Agent 工具调用中的错误经验沉淀到 MySQL，并通过 C++ 向量检索服务在任务执行前召回相似经验注入 Prompt，减少重复工具调用错误和字段幻觉。

## 项目 Bullet

- 设计 Agent 错误经验记忆闭环：在工具调用失败时通过 Python Client SDK 记录任务类型、用户问题、工具名、错误信息、调用上下文等数据，并将可复用经验沉淀为结构化记忆。
- 使用 MySQL 作为长期存储，维护错误日志、经验记忆和向量数据表，支持经验的持久化保存、查询、命中次数更新和后续索引加载。
- 实现 C++ 内存向量检索服务，提供 HTTP 检索接口，基于 cosine similarity 对经验向量进行 top-k 相似度搜索，用于快速召回与当前任务相关的历史经验。
- 封装 Python Client SDK，提供错误记录、经验检索、向量索引加载和 Prompt Context 构造能力，使上层 Agent 可以以轻量方式接入经验记忆能力。
- 将系统最小化集成到 CSV-Insight-Agent：Agent 执行前根据用户 query 检索相似经验并注入 Prompt，同时在图表生成等工具异常时记录错误上下文，帮助减少字段名错误、工具参数错误等重复问题。

## 面试口述版介绍

这个项目是我为 AI Agent 做的一套经验记忆检索系统，核心目标是让 Agent 不只是每次失败后返回错误，而是把工具调用中的错误沉淀成可复用经验。系统里我用 MySQL 存储错误日志、经验记忆和向量数据，用 Python Client SDK 负责错误记录、经验查询、embedding 生成和 Prompt Context 构造；同时实现了一个 C++ 内存向量检索服务，通过 cosine similarity 做 top-k 检索。实际接入时，我把它和 CSV-Insight-Agent 做了最小化集成：在 Agent 执行任务前，会根据用户问题检索相似历史经验，并把经验提醒注入到 Prompt 中；如果图表生成等工具调用发生异常，也会记录工具参数、字段信息和错误上下文。这样可以减少类似字段名写错、工具参数幻觉、重复调用失败等问题，同时不影响原项目在记忆服务不可用时的正常运行。
